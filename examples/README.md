# Examples

Sprint 1 provides `weekly_intake_example.json` as the stable sample intake artifact.

Expected Excel structure for `load_intake_excel.m`:
- `Metadata` (single-row key/value material)
- `Activities`
- `DocumentsReviewed`
- `Meetings`
- `Risks`
- `Issues`
- `Actions`
- `NextSteps`

Excel generation is intentionally deferred to Sprint 2 in this environment to avoid format/tooling fragility.
